# Rust LSP Capability Parity Audit

Step 6A audit for `codex/revolution`.

This file tracks the Rust server capabilities against the current stdio/package
test evidence. Treat "weak" entries as the next Step 6B/6C queue: either prove
the behavior with a focused test, document that the advertised behavior is
intentionally empty, or implement the missing responder.

## Current Evidence

- Branch state when this audit was started: `codex/revolution` clean and synced
  with `origin/codex/revolution`.
- Rust capabilities are declared in `crates/asp-lsp-server/src/main.rs` by
  `server_capabilities`.
- Stdio coverage is concentrated in `crates/asp-lsp-server/tests/stdio.rs`.
- VSIX/cutover packaging coverage is concentrated in
  `apps/vscode/test/package.test.ts`.
